# 📌 What is Routing?

Routing is the process of mapping incoming HTTP requests to specific handler functions on the server based on:
- **URL (Path)**
- **HTTP Method** (GET, POST, PUT, DELETE, etc.)

> 👉 **In simple terms:** Routing decides “which code should run when a request comes in.”

---

# 🧠 How Requests Find Their Way (Request Flow)

When a client (browser/app) sends a request:
`Client → Server → Router → Controller → Service/Database → Response`

## 🔄 Step-by-step:
1. **Client sends request:**
   ```http
   GET /api/users
   ```
2. **Server receives request**
3. **Router checks:**
   - Path → `/api/users`
   - Method → `GET`
4. **Finds matching route**
5. **Calls corresponding handler** (controller function)
6. **Processes data** (if needed)
7. **Sends response back to client**

---

# ⚙️ Example (Node.js + Express)

```javascript
app.get('/api/users', (req, res) => {
  res.json({ message: "All users" });
});
```
> 👉 *When user visits `/api/users`, this function runs.*

---

# 📂 Types of Routing

## 1. Static Routing
Fixed URL paths.
```javascript
app.get('/about', (req, res) => {
  res.send("About Page");
});
```

## 2. Dynamic Routing (Route Parameters)
```javascript
app.get('/user/:id', (req, res) => {
  res.send(req.params.id);
});
```
> 👉 `/user/101` → `id = 101`

## 3. Query-based Routing
```javascript
app.get('/search', (req, res) => {
  console.log(req.query.name);
});
```
> 👉 `/search?name=smriti`

## 4. Nested Routing
```javascript
app.use('/api', apiRoutes);
```
> 👉 Groups routes under a common prefix.

## 5. Modular Routing
```javascript
const router = require('express').Router();

router.get('/users', handler);

module.exports = router;
```
> 👉 Helps in organizing large applications.

## 6. Middleware-based Routing
```javascript
app.get('/dashboard', authMiddleware, (req, res) => {
  res.send("Dashboard");
});
```
> 👉 Middleware runs before the main handler.

# 🌐 HTTP Methods (Core of Routing)

| Method | Purpose |
|---|---|
| **GET** | Retrieve data |
| **POST** | Create new data |
| **PUT** | Update entire data |
| **PATCH** | Update partial data |
| **DELETE** | Remove data |

---

# 📦 Request Data Handling

| Property | Description |
|---|---|
| `req.params` | URL parameters (`/user/:id`) |
| `req.query` | Query string (`?name=abc`) |
| `req.body` | Data sent in request body |

---

# 🏗️ RESTful Routing (Best Practice)

| Action | Route | Method |
|---|---|---|
| Get all users | `/users` | `GET` |
| Get one user | `/users/:id` | `GET` |
| Create user | `/users` | `POST` |
| Update user | `/users/:id` | `PUT` |
| Delete user | `/users/:id` | `DELETE` |

# ⚠️ Route Matching Priority

Routes are matched top to bottom.
```javascript
app.get('/user/profile', ...)
app.get('/user/:id', ...)
```
> 👉 **Wrong order can cause unexpected behavior.** For example, `/user/:id` will match `/user/profile` if placed first. Put specific routes before dynamic parameters.

---

# 🔐 Middleware in Routing

Middleware functions are used for:
- Authentication
- Authorization
- Logging
- Validation

```javascript
app.use('/api', authMiddleware, apiRoutes);
```

---

# ❌ Common Errors

- **404 Not Found** → Route doesn’t exist
- **405 Method Not Allowed** → Wrong HTTP method
- **Incorrect route order**
- **Missing middleware**

---

# 🎯 Key Takeaways

- ✅ Routing connects **request → handler**
- ✅ Based on **URL + HTTP method**
- ✅ Supports **parameters, queries, middleware**
- ✅ Essential for building **REST APIs**
- ✅ Improves code **organization and scalability**
