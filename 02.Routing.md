# рЯУМ What is Routing?

Routing is the process of mapping incoming HTTP requests to specific handler functions on the server based on:
- **URL (Path)**
- **HTTP Method** (GET, POST, PUT, DELETE, etc.)

> рЯСЙ **In simple terms:** Routing decides вАЬwhich code should run when a request comes in.вАЭ

---

# рЯІ† How Requests Find Their Way (Request Flow)

When a client (browser/app) sends a request:
`Client вЖТ Server вЖТ Router вЖТ Controller вЖТ Service/Database вЖТ Response`

## рЯФД Step-by-step:
1. **Client sends request:**
   ```http
   GET /api/users
   ```
2. **Server receives request**
3. **Router checks:**
   - Path вЖТ `/api/users`
   - Method вЖТ `GET`
4. **Finds matching route**
5. **Calls corresponding handler** (controller function)
6. **Processes data** (if needed)
7. **Sends response back to client**

---

# вЪЩпЄП Example (Node.js + Express)

```javascript
app.get('/api/users', (req, res) => {
  res.json({ message: "All users" });
});
```
> рЯСЙ *When user visits `/api/users`, this function runs.*

# рЯУВ Types of Routing

## 1. Static Routing
Fixed URL paths.
```javascript
app.get('/about', (req, res) => {
  res.send("About Page");
});
```

## 2. Dynamic Routing (Route Parameters)
```javascript
app.get('/user/:id', (req, res) => {
  res.send(req.params.id);
});
```
> рЯСЙ `/user/101` вЖТ `id = 101`

## 3. Query-based Routing
```javascript
app.get('/search', (req, res) => {
  console.log(req.query.name);
});
```
> рЯСЙ `/search?name=smriti`

## 4. Nested Routing
```javascript
app.use('/api', apiRoutes);
```
> рЯСЙ Groups routes under a common prefix.

## 5. Modular Routing
```javascript
const router = require('express').Router();

router.get('/users', handler);

module.exports = router;
```
> рЯСЙ Helps in organizing large applications.

## 6. Middleware-based Routing
```javascript
app.get('/dashboard', authMiddleware, (req, res) => {
  res.send("Dashboard");
});
```
> рЯСЙ Middleware runs before the main handler.

# рЯМР HTTP Methods (Core of Routing)

| Method | Purpose |
|---|---|
| **GET** | Retrieve data |
| **POST** | Create new data |
| **PUT** | Update entire data |
| **PATCH** | Update partial data |
| **DELETE** | Remove data |

# рЯУ¶ Request Data Handling

| Property | Description |
|---|---|
| `req.params` | URL parameters (`/user/:id`) |
| `req.query` | Query string (`?name=abc`) |
| `req.body` | Data sent in request body |

# вЪ†пЄП Route Matching Priority

Routes are matched top to bottom.
```javascript
app.get('/user/profile', ...)
app.get('/user/:id', ...)
```
> рЯСЙ **Wrong order can cause unexpected behavior.** For example, `/user/:id` will match `/user/profile` if placed first. Put specific routes before dynamic parameters.

# рЯФР Middleware in Routing
